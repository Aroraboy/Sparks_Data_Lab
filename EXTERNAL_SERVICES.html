Executive Summary

SPARKS DataLab requires 10 external services to operate — including hosting, database, AI, and data enrichment providers. The platform replaces manual WhatsApp-based data request workflows with a centralized data intelligence system featuring AI-powered research, contact enrichment, email verification, permit scraping, and automated reporting.

Service Details

1. Vercel Core $0 – $20/mo

Purpose
Frontend hosting — serves the React single-page application to users
Used For
Hosting the production build of the client app (npm run build → dist/), global CDN, HTTPS, preview deployments
Website
https://vercel.com
Pricing
Hobby (Free): 100 GB bandwidth/month, 1 team member, custom domains. Pro: $20/month — unlimited bandwidth, team collaboration, analytics
Recommended
Hobby (Free) is sufficient for internal team use
Setup
Connect GitHub repo → Set root directory to client → Set build command npm run build → Add environment variables (VITE_SUPABASE_URL, VITE_SUPABASE_ANON_KEY)

2. Railway Core $5 – $20/mo

Purpose
Backend hosting — runs the Express API server and background workers as separate services
Used For
Two services: (1) API server (node server/index.js) and (2) Workers process (node workers/index.js), plus Redis plugin for job queues
Website
https://railway.app
Pricing
Trial: $5 one-time credit. Hobby: $5/month — 8 GB RAM, 8 vCPU, 100 GB bandwidth. Pro: $20/month — production scaling, priority support
Recommended
Hobby ($5/month) for initial deployment; upgrade to Pro for production traffic
Includes
Redis plugin (built-in, no separate account needed), environment variable management, deploy-on-push from GitHub
Setup
Create project → Add 3 services: (1) Express server (root dir server), (2) Workers (root dir workers), (3) Redis plugin → Add all .env variables → Deploy

3. Supabase Core $25/mo

Purpose
PostgreSQL database hosting, user authentication (Google OAuth), Row Level Security
Used For
All data storage — requests, datasets, contacts, permits, notifications, users; Google sign-in for team members
Website
https://supabase.com

Recommended

Keys Required

⚠ Security & Architecture Notes

  • No data leaves the platform — all API calls are server-side only; no API keys are exposed to the browser.
  • Rate limiting is built in — all batch jobs include delays (100–200ms) to stay within API rate limits.
  • Graceful degradation — features that depend on optional services will show errors but won't crash the app if keys are missing.
  • All services use HTTPS — encrypted in transit.
  • Supabase Google Auth — requires a separate Google OAuth client configured in the Supabase dashboard for team member sign-in.
