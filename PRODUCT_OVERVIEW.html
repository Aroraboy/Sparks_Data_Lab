What Is SPARKS DataLab?

SPARKS DataLab is a private internal web application built exclusively for the SPARKS group of companies. It is a centralized data intelligence platform that replaces the current manual workflow — where team members send data requests via WhatsApp and chat messages — with an organized, AI-powered system that handles everything in one place.

In simple terms: Team members log in, submit a data request (e.g., "Find me all commercial general contractors in DFW"), and the platform automatically runs AI research, enriches contacts with real emails and phone numbers, scrapes public permit data, verifies email deliverability, and stores everything in a searchable repository — all without leaving the app.

Who Is It For?

The platform serves all five SPARKS group companies and their teams:

🏗️ TX Sparks Construction 🔨 SuperConstruct 🏠 REF — Real Estate Forum 🏢 Leezaspace ⚙️ General Operations

Each team member gets a personal login (Google sign-in with company email). Access is restricted to SPARKS team members only — no external users can access the platform.

The Problem We're Solving

❌ Current Workflow (Manual)

  • Team members request data through WhatsApp messages and group chats — unstructured, no tracking
  • Requests get lost, forgotten, or duplicated because there's no central system
  • No way to know the status of a request — is it being worked on? Is it done?
  • Research data is scattered across spreadsheets, emails, and chat threads
  • Contact information is manually gathered — time-consuming, often incomplete or outdated
  • Permit data from city portals is checked manually — easy to miss new filings
  • No single repository or database to search past research

✅ With SPARKS DataLab

  • All data requests go through a structured form — titled, prioritized, assigned, tracked
  • Every request has a status trail — submitted → in progress → completed — visible to everyone
  • AI automatically researches queries using Claude (Anthropic) — returns structured, actionable intelligence
  • Contacts are enriched automatically with real emails, phone numbers, LinkedIn profiles, job titles
  • Email addresses are verified for deliverability before outreach
  • City permit portals are scraped automatically every morning — new commercial permits surface without manual checking
  • Everything is stored in a searchable central repository — datasets, contacts, permits, research history
  • Team gets automated email reports — weekly status digest, overdue request alerts

Platform Features

📋

Request Portal

Submit structured data requests with type, priority, company, market, and deadline. Track status from submission to completion. Add comments and collaborate.

How It Works — User Flow

1. Team member logs in

Google sign-in with company email. Lands on the dashboard showing recent activity and stats.

2. Submits a data request

Fills out a structured form — title, description, type, company, market, priority, deadline. System checks for duplicate requests.

3. Request gets assigned & tracked

Assigned team members get an email notification. Request appears on everyone's dashboard with real-time status.

4. AI research runs

The research engine processes the query — extracts intent, runs AI analysis, returns a formatted report with actionable data.

5. Contacts get enriched & verified

Found contacts are enriched with real emails, phones, LinkedIn profiles. Emails are verified for deliverability.

6. Data stored in repository

All research output, contacts, and permits go into searchable datasets. Available for future reference — nothing gets lost.
